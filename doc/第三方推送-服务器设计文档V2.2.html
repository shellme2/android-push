三、推送调用

1. 推送API

两个API功能一样，二选一。
POST {userName}/{appName}/push
POST /push

2. 功能说明

传递一个推送JSON对象，向多个别名、标签推送一条消息，即单播、组播、广播共用此API。

3. 推送JSON对象

包含了词条消息的受众（audience），内容（msg），消息ID（msgId）。

{
    "userName":xtc,
    "appName":xtcdhsb,
    "module":"module-name",     
    "broadcast":false,
    "audience":{
        "alias":["alias1","alias2"],
        "tag":["tag1","tag2"],
        "tagAnd":["tag3","tag4"]
    },
    "msg":"这里的内容会被推送给相应的别名",
    "msgId":"3e41b26692af11e5ab0cecf4bbdf1a90", 
    "options":{
        "offlineUnpush":true
    }
}

① 各字段参数解释

userName：必填，IM管理平台登陆用户名。
appName：必填，管理系统创建的应用名称（注意：区分应用名和包名）。
module：选填，消息所属的业务模块，用于将消息推送到App的指定模块。
broadcast：与audience二选一，是否广播消息，默认fasle。
audience：与broadcast二选一，消息受众。

别名：是由应用生成的length<45的字符串，别名是推送对象的基本单位，在同一应用中，别名是唯一的，别名可以在多个设备上登录，但同一时间，别名只能在一个设备上登录。在一次推送中，同一个别名只会收到一条推送消息。
标签：是由应用生成的length<45的字符串，标签是一组别名的统一标识，给一个标签推送消息相当于给这组别名推送消息。

字段名 字段类型 说明 备注
alias Array[String] 别名，取并集 数组长度不超过1000
tag Array[String] 标签，取并集 数组长度不超过1000，给所有含有这组标签中至少一个的别名推送消息，即这组标签下的所有别名。
tagAnd Array[String] 标签，取交集 数组长度不超过1000，给所有含有这组标签的别名推送消息。
  1. alias、tag、tagAnd可以并存，并存时各项之间取交集。
  2. 当同时存在两项以上不为null时，若有一项没有对应的别名，那么本次调用不回给任何别名推送消息。
options：选填，包含其他可选的扩展功能字段。
字段名 字段类型 默认值 说明 备注
offlineUnpush Boolean false 该条消息是否不推送给离线设备 设置为true时，该条消息不会推送给当前离线的设备，当离线设备上线时也不回收到该条消息。
